Retrieve Data from Parameters and Assign to Screen Fields

  1. In the Instant App designer, make sure that the Layouts tab is expanded.
  2. Click Events and Actions and then App Sent.
  3. For each layout component (except image_header and btnSubmit):
    • Click Add Action and select Set Element Value.
    • Mouse over the layout component, press Ctrl, and then drag the component to Drag and drop elements from the Layout field in the tile for the action.
    • In the New Value field, fill in the corresponding parameter and surround it with curly braces.

    For example, for the inputSenderName element, you'd fill in {senderName} as the value.

    4. Troubleshooting: If, when trying to drag the component, you accidentally select it instead, follow do the following to get back to where you were:

    • In the left navigation of the designer, scroll down and select Events and Actions, and then click App Sent.

Integrate the Verify_Order Instant App with the Bot

Now that the instant app is ready, we'll go to the Bot designer and integrate our bot with this instant app.

  1. Click and select Development > Bots.
  2. (If it's not already open) open advt24hrsflowers_bot6<YourUniqueInitials>
  3. In the left navigation, click to open the dialog flow editor.
  4. Replace the editOrder state with this code to call the instant app: 
    # START - Instant Apps - Edit Order is being implemented now.  
#  editOrder:
#    component: "System.Output"
#    properties:   
#      text: "Sorry. Editing orders is a feature under development"
#      keepTurn: true
#    transitions:
#      return: "submitOrder"

  editOrder:
    component: "System.Interactive"
    properties:
      sourceVariableList: "senderName,senderMessage,orderQuantity,orderProductName,recipientName,recipientAddress,flowerCost,deliveryCost"
      variable: "payment" 
      id: "Verify_24_Hrs_Order_XX"
      prompt: "Please click the link below to make changes otherwise cancel"
      linkLabel: "Make Changes"
      cancelLabel: "No Changes"
      translate: 
    transitions:
      next: "verifiedDataCopyBack"
      actions:
        cancel: "paymentOrCancel"
        textReceived: "paymentOrCancel"
  5. Below that, add this state to copy data from the instant app to the bot variables. 
      verifiedDataCopyBack:
    component: "System.CopyVariables"
    properties:
      from: "payment.value.senderName,payment.value.senderMessage,payment.value.orderQuantity,payment.value.orderProductName,payment.value.recipientName,payment.value.recipientAddress,payment.value.flowerCost,payment.value.deliveryCost"
      to: "senderName,senderMessage,orderQuantity,orderProductName,recipientName,recipientAddress,flowerCost,deliveryCost"
    transitions:
      next: "displayVerifiedOrder"
  6. And add this state to display the verified order: 
      displayVerifiedOrder:
    component: "System.Output"
    properties:       
      text: "Your order ${orderNumber.value} for ${orderQuantity.value.number} item(s) of  ${orderProductName.value} is ready for delivery.We deliver to ${recipientName.value}, ${recipientAddress.value}"
      keepTurn: true
    transitions:
      next: "paymentOrCancel"

Make a Payment

Now we'll add the logic to check whether the customer wants to make a payment or cancel the order.

  1. In the left navigation, click to open the dialog flow editor.
  2. After the displayVerifiedOrder add a new System.List component to check whether customer would like to make a payment and place the order:
      paymentOrCancel:
    component: "System.List"
    properties:       
      prompt: "Do you want to make a payment and submit this order?"
      options:
        - label: "Yes, please submit"
          value: "submit"
        - label: "No, please cancel"
          value: "cancel"  
    transitions:
      actions:
        submit: "setCustomerName"
        cancel: "cancelOrder"
  3. Add the setCustomerName and setFlowerCoststates to check if the customer wants to make a payment, capture the customer name and the cost of the order, and send that info to another instant app. 
      setCustomerName:
    component: "System.SetVariable"
    properties:
      variable: "customerName"
      value: "${profile.firstName} ${profile.lastName} "
    transitions:
      next: "setFlowerCost"
      
  setFlowerCost:
    component: "System.SetVariable"
    properties:
      variable: "flowerCost"
      value: "169"
    transitions:
      next: "makePayment"

Add Actions and Events

Now we'll add a few events and actions. Let's start with an event to populate the customer name.

  1. In the Instant App designer, make sure that the Layouts tab is expanded.
  2. Click Events and Actions and then App Sent.
  3. Click Add Action and select Set Element Value.
  4. Mouse over the input_name layout component, press Ctrl, and then drag the component to the Drag and drop elements from the Layout field.
  5. In the New Value field, enter {customerName}.

Now we'll add an event for the Reset button.

  1. Click the Events and Actions tile again.
  2. In the middle pane of the designer, scroll down and select the reset Pressed tile.
  3. From the Next dropdown, select button is pressed regardless of validation.
  4. Click Add Action.
  5. Click the Reset Elements tile.
  6. In the Layout part of the left navigation, mouse over the validate component, press Ctrl, and then drag it to the Drag and Drop an Element or Pane from the Layout field in the tile for the action.
  7. Repeat the above step for the following components:
    • input_card_number
    • input_exp_date
    • input_verifi_code
    • messageKO

Now let's add an event for the Validate button. For this event, we'll call an external API to validate the credit card and process the return value using a local function.

  1. Click the Events and Actions tile again.
  2. In the middle pane of the designer, scroll down and select the validate Pressed tile.
  3. Click Add Action and select Call External Web API.
  4. Fill in the following values for the action:
    • Method: POST
    • URL: http://infra.digitalpracticespain.com:9100/cc/validate
    • Content Type: application/json
    • Value: {"creditcard": {input_card_number}}
    • Data Key: myData
    • Event Name: validateAPI
    • Test Response Status Code: 200
    • Test Response: {"valid": true}

Now we need to provide a JavaScript function for the validate API.

  1. Click Events and Actions, and then App Sent.
  3. In the middle pane of the designer, scroll down to the bottom and select Add Named Callback Event.
  4. For Event Name, fill in validateAPI.
  5. In the Action text area, paste the following function: 
    var result = JSON.parse(returnData);

//Skip Validation
var chkValid = result.valid;
chkValid = true; // Currently hard coded to return “true” always.

if (chkValid)
{   
   app.hideElements("messageKO");
   app.hideElements("Validate");
   app.hideElements("reset");
   app.showElements("button_submit");
   app.showElements("messageOK");
}else if (!chkValid){
   app.showElements("messageKO");
   app.showAlertDialog("Your credit card number is not valid. Please try again. For example use: 4444333322221111");
   //app.showAlertDialog("tarjeta: "+app.elements.singleLineInput_1);    
   //app.elements.singleLineInput_1.value=0;
}

Integrate the Flowers Payment Instant App with the Bot

....

  1. Return to the bot designer and navigate to your bot (advt24hrsflowers_bot6<YourUniqueInitials>).
  2. Click to open the dialog flow editor.
  3. Navigate to the setFlowerCost state and paste the following two states below it: 
      makePayment:
    component: "System.Interactive"
    properties:
      sourceVariableList: "flowerCost, delieryCost, customerName"
      variable: "payment" 
      id: "24_Hrs_Flowers_Payment_XX"
      prompt: "To make a payment click the Make Payment link below"
      linkLabel: "Make Payment"
      cancelLabel: "Cancel"
      translate: 
    transitions:
      next: "beforeSubmit"
      actions:
        cancel: "cancelOrder"
        textReceived: "beforeSubmit"   
  
  beforeSubmit:
    component: "System.Output"
    properties:
      text: "Your payment has been received, please wait..."
      keepTurn: true
    transitions:
      next: "submitOrder"
      
# END - Instant Apps - Edit Order is being implemented now.

    The makePayment state is a System.Interactive component that calls the Flowers Payment instant app.

    It calls beforeSubmit to display a message to the user and use keepTurn: true to jump to submitOrder. At the moment this simply displays a message that the order has been submitted. In future, you could add a call to an API to process the order.

  4. In the orderSummary state, change the value of the SubmitOrder attribute to "paymentOrCancel".

    This connects the orderSummary screen to the payment instant app so that, assuming there are no further changes, the customer can make their purchase.
